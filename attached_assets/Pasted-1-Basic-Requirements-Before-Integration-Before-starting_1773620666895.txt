Before starting the integration, make sure the following credentials are available from PaySprint.

API Credentials

Usually PaySprint provides:

JWT_KEY – used to generate authentication tokens

AES_ENCRYPTION_KEY – used to encrypt request data

AES_ENCRYPTION_IV – initialization vector for encryption

Merchant / Partner ID

Callback URL

Whitelisted IP address

Server Requirement

Recharge APIs must be called from a backend server, not directly from the frontend.

Supported backend environments:

Node.js

PHP

Java

Python

Important:
Never call recharge APIs directly from JavaScript in the browser.

4. Generate JWT Token

PaySprint APIs use JWT authentication.

The token is generated using:

JWT_KEY
Example (Node.js)
const jwt = require("jsonwebtoken");

function generateToken() {
    const payload = {
        timestamp: Date.now()
    };

    const token = jwt.sign(payload, "YOUR_JWT_KEY");
    return token;
}

Best practice: generate a new token for each API request.

5. AES Encryption of Request Data

PaySprint requires the request payload to be encrypted using:

AES_ENCRYPTION_KEY
AES_ENCRYPTION_IV
Example (Node.js)
const crypto = require("crypto");

function encryptData(data) {

    const key = Buffer.from("AES_KEY");
    const iv = Buffer.from("AES_IV");

    const cipher = crypto.createCipheriv("aes-256-cbc", key, iv);

    let encrypted = cipher.update(JSON.stringify(data), "utf8", "base64");
    encrypted += cipher.final("base64");

    return encrypted;
}

When a user performs a recharge, the backend prepares a request.

Example request:

{
  mobile: "9876543210",
  operator: "AT",
  amount: "199",
  referenceid: "TXN123456"

Field explanation:

Field	Description

8. Example Recharge API Call (Node.js)
const axios = require("axios");

async function recharge() {

    const requestData = {
        mobile: "9876543210",
        operator: "AT",
        amount: "199",
        referenceid: "TXN12345"
    };

    const encrypted = encryptData(requestData);

    const token = generateToken();

    const response = await axios.post(
        "https://api.paysprint.in/api/v1/recharge",
        {
            data: encrypted
        },
        {
            headers: {
                Authorization: "Bearer " + token
            }
        }
    );

    console.log(response.data);
}

10. Callback URL Handling

After recharge processing, PaySprint sends the final status to your callback URL.

Example callback endpoint:

https://yourdomain.com/recharge/callback

Example callback response:

{
  referenceid: "TXN12345",
  status: "SUCCESS"
}

Possible status values:

SUCCESS
FAILED
PENDING
11. Example Callback API (Node.js)
app.post("/recharge/callback", (req, res) => {

    const reference = req.body.referenceid;
    const status = req.body.status;

    if (status === "SUCCESS") {
        updateTransaction(reference, "SUCCESS");
    }

    if (status === "FAILED") {
        updateTransaction(reference, "FAILED");
    }

    res.send("OK");
});

This endpoint updates the transaction status in the database.

15. Security Best Practices

Recharge APIs deal with financial transactions, so security is very important.

Always follow these practices:

Always call APIs from the server

Generate unique reference IDs

Enable IP whitelisting

Encrypt request data

Verify callbacks

Maintain complete transaction logs

16. Production Deployment Checklist

Before launching the recharge service, confirm:

Server IP is whitelisted

Callback URL is reachable

JWT token generation works

AES encryption works correctly

Transactions are saved in the database

Error logging is implemented
